- `User/ds1302.h` - DS1302头文件
- `User/ds1302.c` - DS1302驱动实现

### 2. 测试文件
- `User/main.c` - OLED显示DS1302时间测试程序

## 硬件接线

### OLED接线（项目中实际配置）
```
OLED模块    →  STM32F103C8T6
────────────────────────────
VCC         →  3.3V 或 5V
GND         →  GND
SCL         →  PB8
SDA         →  PB9
（RES和DC由模块内部处理）
```

### DS1302接线
```
DS1302模块  →  STM32F103C8T6
────────────────────────────
VCC         →  3.3V 或 5V
GND         →  GND
CLK (SCLK)  →  PA5
DAT (I/O)   →  PA6
RST (CE)    →  PA4
```

**注意**：请确保插入CR2032纽扣电池到DS1302模块的电池座！

## 使用方法

### 步骤1：复制驱动文件
将以下文件复制到你的工程中：
- `ds1302.h` → 工程 HARDWARE/DS1302/ 目录
- `ds1302.c` → 工程 HARDWARE/DS1302/ 目录

### 步骤2：复制测试文件
将 `test_ds1302.c` 复制到工程 USER 目录

### 步骤3：在Keil中添加文件
1. 打开Keil工程
2. 在Project窗口中右键 HARDWARE 文件夹
3. 选择 "Add Existing Files to Group..."
4. 添加 `HARDWARE/DS1302/ds1302.c`
5. 在USER文件夹中添加 `test_ds1302.c`

### 步骤4：修改main.c
在 `main.c` 中调用测试函数：

```c
#include "sys.h"
#include "delay.h"

extern void DS1302_Test(void);

int main(void)
{
    delay_init();
    DS1302_Test();

    while(1)
    {
    }
}
```

### 步骤5：编译并下载
1. 编译工程
2. 下载到STM32开发板
3. 观察OLED显示

## 测试结果判断

### ✅ 成功
- OLED显示 "DS1302 OK!"
- 时间每秒更新（秒数递增）
- 日期显示正确
- 按设定的时间运行

### ❌ 失败
- OLED显示 "Init DS1302..." 后无变化
  - 检查DS1302接线
  - 检查电池是否安装
  - 检查供电是否正常

### ⚠️ 常见问题

#### 问题1：OLED不显示
**检查**：
- OLED接线（PB6, PB7, PB0, PB1）
- OLED初始化代码
- OLED供电

#### 问题2：DS1302初始化后无响应
**检查**：
- DS1302接线（PA4, PA5, PA6）
- 电池是否插入
- 电池电压是否正常（应>2.5V）

#### 问题3：时间不正确
**原因**：
- 首次上电，DS1302内部时间可能随机
- 测试程序会设置默认时间：2026-04-09 12:00:00
- 如果断电后时间归零，说明电池没电或接触不良

## 高级功能

### 读取当前时间
```c
DS1302_TimeTypeDef time;
DS1302_GetTime(&time);

printf("Time: %02d:%02d:%02d\r\n",
       time.hour, time.minute, time.second);
printf("Date: 20%02d-%02d-%02d\r\n",
       time.year, time.month, time.day);
```

### 设置时间
```c
DS1302_TimeTypeDef time;
time.year = 26;
time.month = 4;
time.day = 9;
time.hour = 12;
time.minute = 30;
time.second = 0;
time.week = 4;

DS1302_SetTime(&time);
```
