# DS1302时钟模块接线指南

## 一、与STM32内部RTC的区别

### 1. STM32内部RTC（系统时钟）

**工作原理**：
```
┌─────────────────────────────────────────────────────────┐
│                   STM32 内部结构                          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   ┌─────────────────────────────────────────────┐       │
│   │              STM32F103C8T6                    │       │
│   │  ┌───────────────────────────────────────┐  │       │
│   │  │  72MHz 主时钟                          │  │       │
│   │  │  ┌─────────────────────────────────┐  │  │       │
│   │  │  │                                 │  │  │       │
│   │  │  │    ┌─────────────────────┐     │  │  │       │
│   │  │  │    │   RTC 实时时钟单元    │     │  │  │       │
│   │  │  │    │   (需要备用电源)      │     │  │  │       │
│   │  │  │    └─────────────────────┘     │  │  │       │
│   │  │  │                                 │  │  │       │
│   │  │  └─────────────────────────────────┘  │  │       │
│   │  └───────────────────────────────────────┘  │       │
│   │  ↑                                        │       │
│   │  └─ 需要外接 32.768kHz 晶振（可选）        │       │
│   └─────────────────────────────────────────────┘       │
│                                                          │
│   ⚠️ 问题：主电源断电 → RTC停止 → 时间归零               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**特点**：
- ✅ 集成在STM32内部
- ✅ 不需要额外购买模块
- ❌ 断电后时间丢失（除非VBAT接备用电源）
- ❌ 精度相对较低（温漂较大）
- ⚠️ 配置较复杂

### 2. DS1302外部时钟模块

**工作原理**：
```
┌─────────────────────────────────────────────────────────┐
│                   DS1302 模块结构                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   ┌───────────────────────────────────────┐             │
│   │         DS1302 模块                    │             │
│   │                                        │             │
│   │    ┌─────────────────────────────┐    │             │
│   │    │        DS1302 芯片           │    │             │
│   │    │   ┌─────────────────────┐   │    │             │
│   │    │   │   31× 时钟/日历     │   │    │             │
│   │    │   │   写保护寄存器      │   │    │             │
│   │    │   │   31× RAM          │   │    │             │
│   │    │   └─────────────────────┘   │    │             │
│   │    └─────────────────────────────┘    │             │
│   │                                        │             │
│   │    [32.768kHz晶振]                    │             │
│   │    [CR2032电池座] 🔋                   │             │
│   │                                        │             │
│   └───────────────────────────────────────┘             │
│                                                          │
│   ✅ 独立运行 → 主电源断电 → 纽扣电池供电 → 时间继续走     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**特点**：
- ✅ 自带电池座（CR2032纽扣电池）
- ✅ 断电后由电池供电，时间继续走
- ✅ 精度高（32.768kHz专用晶振）
- ✅ 独立于MCU运行
- ❌ 需要额外购买（约5-10元）
- ❌ 占用GPIO引脚

## 二、两种时钟对比

| 特性 | STM32内部RTC | DS1302模块 |
|------|-------------|-----------|
| **成本** | 免费（内置） | 约5-10元 |
| **购买** | 不需要 | 需要购买 |
| **接线** | 简单（只需晶振） | 需要4-5根线 |
| **电池** | 需要外接VBAT | 自带电池座 |
| **断电保持** | ❌ 需要额外电路 | ✅ 自动保持 |
| **精度** | 一般（温漂） | 高（专用晶振） |
| **复杂度** | 配置复杂 | 使用简单 |
| **代码量** | 需要配置 | 有现成库 |

## 三、为什么智能药箱需要DS1302？

### 使用场景对比

#### 没有DS1302（只用STM32内部RTC）：
```
💊 用户设置8:00吃药
💻 MCU运行中...
🔌 突然断电！
⏰ 8小时后恢复供电
💻 MCU重启 → RTC归零 → 8:00变成00:00？
💊 吃药提醒：8:00（错误的时间！）
```

#### 有DS1302：
```
💊 用户设置8:00吃药
💻 MCU运行中...
🔌 突然断电！
🔋 DS1302的CR2032电池继续供电
⏰ 8小时后恢复供电
💻 MCU重启
🔋 DS1302正确时间：16:00
💊 立即提醒：16:00吃药！
```

## 六、GPIO配置代码

### HAL库配置

```c
// DS1302引脚定义
#define DS1302_RST_PIN    GPIO_PIN_4   // PA4 - 复位/片选
#define DS1302_CLK_PIN    GPIO_PIN_5   // PA5 - 时钟
#define DS1302_DAT_PIN    GPIO_PIN_6   // PA6 - 数据

#define DS1302_PORT       GPIOA

void DS1302_GPIO_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStruct = {0};

    // 使能GPIOA时钟
    __HAL_RCC_GPIOA_CLK_ENABLE();

    // 配置RST和CLK为推挽输出
    GPIO_InitStruct.Pin = DS1302_RST_PIN | DS1302_CLK_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStruct.Pull = GPIO_NOPULL;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(DS1302_PORT, &GPIO_InitStruct);

    // 配置DAT为推挽输出（默认）
    GPIO_InitStruct.Pin = DS1302_DAT_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    HAL_GPIO_Init(DS1302_PORT, &GPIO_InitStruct);

    // 初始化引脚状态
    HAL_GPIO_WritePin(DS1302_PORT, DS1302_RST_PIN, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(DS1302_PORT, DS1302_CLK_PIN, GPIO_PIN_RESET);
    HAL_GPIO_WritePin(DS1302_PORT, DS1302_DAT_PIN, GPIO_PIN_RESET);
}
```

## 八、基础驱动代码

### 写入一个字节

```c
void DS1302_WriteByte(uint8_t dat)
{
    uint8_t i;

    // 配置DAT为输出
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Pin = DS1302_DAT_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_OUTPUT_PP;
    GPIO_InitStruct.Pull = GPIO_NOPULL;
    GPIO_InitStruct.Speed = GPIO_SPEED_FREQ_LOW;
    HAL_GPIO_Init(DS1302_PORT, &GPIO_InitStruct);

    // 串行发送8位数据
    for(i = 0; i < 8; i++)
    {
        // 设置数据位
        HAL_GPIO_WritePin(DS1302_PORT, DS1302_DAT_PIN,
                         (dat & 0x01) ? GPIO_PIN_SET : GPIO_PIN_RESET);
        dat >>= 1;

        // 产生时钟脉冲
        HAL_GPIO_WritePin(DS1302_PORT, DS1302_CLK_PIN, GPIO_PIN_SET);
        delay_us(1);
        HAL_GPIO_WritePin(DS1302_PORT, DS1302_CLK_PIN, GPIO_PIN_RESET);
        delay_us(1);
    }
}
```

### 读取一个字节

```c
uint8_t DS1302_ReadByte(void)
{
    uint8_t i, dat = 0;

    // 配置DAT为输入
    GPIO_InitTypeDef GPIO_InitStruct = {0};
    GPIO_InitStruct.Pin = DS1302_DAT_PIN;
    GPIO_InitStruct.Mode = GPIO_MODE_INPUT;
    GPIO_InitStruct.Pull = GPIO_NOPULL;
    HAL_GPIO_Init(DS1302_PORT, &GPIO_InitStruct);

    // 串行接收8位数据
    for(i = 0; i < 8; i++)
    {
        // 读取数据位
        if(HAL_GPIO_ReadPin(DS1302_PORT, DS1302_DAT_PIN) == GPIO_PIN_SET)
        {
            dat |= 0x80;  // MSB优先
        }

        // 产生时钟脉冲
        HAL_GPIO_WritePin(DS1302_PORT, DS1302_CLK_PIN, GPIO_PIN_SET);
        delay_us(1);
        HAL_GPIO_WritePin(DS1302_PORT, DS1302_CLK_PIN, GPIO_PIN_RESET);
        delay_us(1);

        dat >>= 1;  // 移位准备接收下一位
    }

    return dat;
}
```

## 九、常见问题

### 问题1：时间不准确
**原因**：晶振质量问题或电容不匹配
**解决**：
- 更换晶振
- 在晶振两端并联15pF电容
- 定期通过网络校准

### 问题2：时间归零
**原因**：电池没电或接触不良
**解决**：
- 更换CR2032电池
- 检查电池座接触
- 测量电池电压（应>2.5V）

### 问题3：读写错误
**原因**：时序不正确
**解决**：
- 确保CLK在RST低电平时稳定
- 读写时序严格按照数据手册
- 降低GPIO翻转速度
