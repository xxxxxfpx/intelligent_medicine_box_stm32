# DS1302 RTC驱动调试记录

## 硬件连接

| STM32引脚 | DS1302引脚 | 功能 |
|-----------|-----------|------|
| PB12 | RST/CE | 片选 |
| PB13 | DAT/I/O | 数据 |
| PB14 | CLK/SCLK | 时钟 |

## 调试过程

### 问题1: 显示 00 00 00

- **原因**: 引脚配置错误，原来使用PA4/PA5/PA6
- **解决**: 更换为PB12/PB13/PB14

### 问题2: 显示 FF FF FF

- **原因**: 通信成功但写入失败
- **解决**: 添加延时，确保时序正确

## 关键发现

1. **DS1302秒寄存器Bit7是时钟暂停位(CH)**
   - CH=1: 时钟停止
   - CH=0: 时钟运行

2. **写入前必须清除写保护位(WP)**
   - 控制寄存器地址: 0x8E(写) / 0x8F(读)
   - WP=0: 允许写入
   - WP=1: 禁止写入

3. **时序需要足够延时**
   - CE to CLK Setup: 4µs (at 2.0V)
   - CLK Low/High Time: 1000ns (at 2.0V)

## DS1302寄存器地址表

| 寄存器 | 写地址 | 读地址 | Bit7 | 范围 |
|--------|--------|--------|------|------|
| 秒 | 0x80 | 0x81 | CH (时钟暂停) | 00-59 |
| 分 | 0x82 | 0x83 | - | 00-59 |
| 时 | 0x84 | 0x85 | 12/24 | 0-23 |
| 日 | 0x86 | 0x87 | - | 1-31 |
| 月 | 0x88 | 0x89 | - | 1-12 |
| 星期 | 0x8A | 0x8B | - | 1-7 |
| 年 | 0x8C | 0x8D | - | 00-99 |
| 控制 | 0x8E | 0x8F | WP (写保护) | - |

## 2026-04-15 调试记录

### 问题3: 秒数跳秒 (1→2→3→4→8)

- **现象**: 秒数显示不连续，出现跳秒，如 21→22→23→24→28→29，刷新速度明显慢于1秒
- **原因**: 读操作时序错误
  - 旧代码在CLK下降沿后读取数据
  - 数据移位方向与DS1302输出顺序不一致
  - 延时时间在临界值，导致不稳定

### 问题4: 代码未改动但出现故障

- **现象**: 几天前正常工作，两天后出现跳秒问题
- **原因**: 代码存在边界条件隐患
  - 旧代码的时序刚好在临界值
  - 环境变化（温度、电压）导致时序对不上
  - "能用"不等于"正确"

### 解决方案

参考51单片机DS1302驱动代码，修正读写时序：

#### 读操作修正

```c
// 旧代码 - 错误
for(i = 0; i < 8; i++) {
    CLK高 → 延时 → CLK低 → 延时 → 读取
    dat |= (0x01 << i);  // 低位优先
}

// 新代码 - 正确
for(i = 0; i < 8; i++) {
    dat = dat >> 1;      // 先右移
    if(读取) dat |= 0x80; // 高位优先（与DS1302输出一致）
    CLK高 → CLK低;       // 无延时
}
```

#### 写操作修正

```c
// 移除所有Delay_us延时
// STM32 GPIO操作速度足够慢，不需要额外延时
for(i = 0; i < 8; i++) {
    if(dat & 0x01) DAT=1; else DAT=0;
    CLK高 → CLK低;
    dat >>= 1;
}
```

### 关键经验

1. **严格遵循参考代码时序**
   - 不能想当然添加延时
   - 数据移位方向必须与芯片输出顺序一致

2. **边界条件代码需充分测试**
   - "能用"不等于"正确"
   - 时序临界值代码可能因环境变化而失效

3. **参考代码对比分析**
   - 51单片机代码经过验证，时序可靠
   - STM32移植时注意GPIO操作速度差异
